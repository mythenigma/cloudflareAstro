# エンタープライズ文書配布：実務で使えるセキュリティ・ワークフロー

<div class="intro-panel">
  <p>企業の文書配布は「PDFを送って祈る」ではありません。アクセス条件を定義し、安全に届け、何が起きたかを記録する——再現性のある運用が必要です。</p>
</div>

![Enterprise document distribution workflow](/diagram/en/enterprise-document-distribution-workflow.svg)

## ワークフロー（図が示していること）

<div class="feature-section">
  <div class="feature-content">
    <h3>1) 準備</h3>
    <ul>
      <li><strong>分類</strong>：公開／社内／機密</li>
      <li><strong>配布対象</strong>：指名制か、リンクを知っている人全員か</li>
      <li><strong>最小権限から開始</strong>：まず厳しめ、必要に応じて緩和</li>
    </ul>
  </div>
  <div class="feature-content">
    <h3>2) 制御</h3>
    <ul>
      <li><strong>本人確認</strong>：メール／電話／パスワードをリスクに合わせて</li>
      <li><strong>制限</strong>：閲覧回数＋有効期限で流出ウィンドウを縮小</li>
      <li><strong>制約</strong>：必要ならダウンロード／印刷を制限</li>
    </ul>
  </div>
</div>

## 仕組み化すべきポイント（スケールさせるために）

- 文書タイプ別の **ポリシーテンプレート**（人事／法務／営業／研修）
- 高リスク文書の **レビュー工程**（二重チェック）
- 緊急時の **停止手段**（即時に無効化できること）

## よくある失敗パターン

| 失敗 | 起きること | 対策 |
| --- | --- | --- |
| 「リンクを知っていれば誰でも」 | 転送でアクセスが拡散 | 本人確認＋有効期限 |
| 永久リンク | 古い版が流通し続ける | 短い期限＋内容更新 |
| 記録がない | 監査・調査ができない | ログ＋エクスポート |
