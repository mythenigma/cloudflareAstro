Ein QR-Code ist letztlich nur ein Bild einer URL. Genau deshalb ist die eigentliche Arbeit nicht "den Code erzeugen", sondern den **richtigen Link hinter dem Code** zu waehlen, den Code so zu platzieren, dass Menschen ihn wirklich scannen, und spaeter im Access Log zu sehen, was aus der Verteilung geworden ist.

![Ein QR-Code ist nur so gut wie der Link und die Regeln dahinter](/maipdf2026/flowchart/en-qr-sharing-flow.svg)

## Warum QR-Code-Verteilung so oft besser funktioniert als ausgeschriebene Links

- Niemand tippt gern lange URLs von einem Flyer ab.
- Ein gedruckter QR-Code funktioniert auf Poster, Verpackung, Visitenkarte oder Messestand gleichermassen.
- Dieselbe Freigabe kann spaeter aktualisiert werden, ohne den gedruckten Code neu zu produzieren.

Genau das ist der grosse Unterschied zu einem statischen PDF-Download, der irgendwo fest eingebaut wurde.

## Wo QR-Verteilung in der Praxis wirklich stark ist

### Gesundheits- und Serviceumfelder

Informationsblaetter, Nachsorgehinweise oder Bedienanleitungen lassen sich vor Ort leichter per Scan oeffnen als in Papierstapeln mitgeben.

### Immobilien und Vor-Ort-Vertrieb

Aushang, Schild oder Expose am Objekt funktionieren als QR-Einstieg meist besser als ausgeschriebene Links.

### Bildung und Events

Kursunterlagen an der Tuere, Workshop-Material auf dem Sitz oder Speaker-Decks auf dem Badge sind klassische QR-Faelle.

### Produktion und Field Service

Sicherheitsblaetter oder Maschinenhandbuecher direkt am Geraet sind per Scan oft schneller erreichbar als ueber ein Intranet-Menue.

## Digital und Print zusammen denken

QR-Codes sind nicht nur fuer Druck. Sie funktionieren auch auf digitalen Flaechen, wenn Leser den Inhalt vom Laptop aufs Handy holen wollen.

- auf Folien oder Livestream-Screens
- in E-Mails, wenn Leser lieber mobil weiterlesen
- in Chat- oder Intranet-Posts

Wichtig ist: Der QR-Code und die URL sollten auf **dieselbe Freigabe** zeigen. Dann haben Sie spaeter nur einen Ort, den Sie verwalten muessen.

## Sinnvolle Schutzprofile fuer QR-Dokumente

### Oeffentliche Infounterlagen

Wenn Reichweite wichtiger ist als Restriktion:

- laengere oder keine Ablaufzeit
- kein Identitaetscheck
- Download je nach Dokument erlaubt

### Partner- oder Kundenunterlagen

Wenn der QR sichtbar ist, aber nicht jeder hinein soll:

- Ablauf setzen
- optional E-Mail-Verifizierung
- bei sensiblen Inhalten Wasserzeichen und view-only

### Interne oder eventbezogene Handouts

Wenn viele Leute scannen, aber nur in einem bestimmten Zeitraum:

- Ablauf auf Veranstaltungsfenster legen
- Oeffnungslimit grosszuegig planen
- bei Bedarf spaeter Freigabe schliessen

## Die Offline-Checkliste vor dem Druck

Die meisten QR-Probleme entstehen nicht im System, sondern beim Druck oder bei schlechter Platzierung.

- **mindestens etwa 2 x 2 cm** drucken
- ausreichend weissen Rand um den Code lassen
- hohe Kontraste verwenden
- den echten Scan auf iPhone und Android testen
- bei groesseren Distanzen im realen Einsatzabstand pruefen
- wenn sinnvoll, eine kurze lesbare URL unter den Code setzen

Ein QR-Code, der auf dem Bildschirm perfekt aussieht, kann auf Papier trotzdem schlecht scannbar sein. Testen vor dem finalen Druck ist Pflicht.

## Was das Access Log nach der Verteilung erzaehlt

Der QR-Code ist nicht nur ein Einstiegspunkt. Er ist auch ein Messpunkt.

- Sie sehen, ob nach dem Druck oder Event tatsaechlich gescannt wurde.
- Sie sehen, ob das Publikum hauptsaechlich mobil oder spaeter am Desktop liest.
- Sie sehen, ob ein eigentlich enger Link ueberraschend weit zirkuliert.

![Mit dem Access Log wird aus QR-Verteilung ein messbarer statt blinder Kanal](/maipdf2026/page_redirect_to_accessRecord.png)

Gerade bei Drucksachen ist das wichtig, weil ohne Access Log fast nie klar ist, ob der verteilte Code ueberhaupt genutzt wurde.

## Was QR-Codes nicht automatisch loesen

- Sie machen einen sensiblen Link **nicht automatisch sicher**.
- Sie reparieren keine schlecht lesbare oder mobil ungeeignete PDF.
- Sie ersetzen keine Zugangskontrolle, wenn der Inhalt begrenzt sein soll.

Die eigentliche Sicherheit liegt immer im Link hinter dem Code, nicht im Code selbst.

## Haeufige Fehler

- QR-Code drucken, ohne den finalen Scan auf echten Geraeten zu testen
- einen sensiblen Link ohne Ablauf oder Zugangskontrolle auf sichtbaren Druckflaechen platzieren
- bei Events mit knappen Oeffnungslimits arbeiten
- nach Datei-Updates neue Links erzeugen, obwohl dieselbe Freigabe weitergenutzt werden koennte

## FAQ

### Bleibt der QR-Code gueltig, wenn ich die Datei spaeter austausche?

Ja, solange Sie die Datei hinter derselben Freigabe ersetzen und nicht eine komplett neue URL erzeugen.

### Wie gross sollte ein gedruckter Code sein?

Als Faustregel mindestens **2 x 2 cm**, bei groesserer Scan-Distanz entsprechend groesser.

### Sollte unter dem QR-Code noch eine URL stehen?

Oft ja. Eine kurze Fallback-URL hilft bei schlechter Beleuchtung oder beschaedigtem Druck.
