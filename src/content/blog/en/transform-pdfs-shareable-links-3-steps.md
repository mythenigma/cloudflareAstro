# Transform PDFs into Shareable Links in 3 Steps

The goal is simple: **Upload → Configure → Share**.

You generate one link (plus QR) and set rules so the link stays **useful** and **controlled**.

## 1) Upload the PDF

Upload starts the workflow. You typically drag the file in or pick it from your device.

![Upload step: header and entry paths to sharing](/maipdf2026/maipdf_header_login_or_upload_file.png)

## 2) Configure access (the part most people skip)

On the **Configure** screen, set:

- **Access limit** (how many opens)
- **Each session** (how long one continuous read can last)
- **Expiration** (preset or custom; includes UTC and local time display)
- Optional for important audiences:
  - **Read alerts (Telegram)**
  - **Email verification**
- Viewing mode:
  - **SecureView** / **FenceView** / **Unrestricted**
- Optional **Dynamic Watermark** (when enabled on your setup)

![Configure screen: access limits, session length, expiry, Telegram, email verification](/maipdf2026/MaiPDF_settings_expiration_telegram.png)

![Big picture flow: upload → secure settings → share link and QR](/maipdf2026/show_off/en-pdf-sharing-workflow-animation.svg)

> Note: if **Access limit** is set above **10,000**, MaiPDF applies unlimited-open behavior and disables access logging, Telegram alerts, and dynamic watermark.

## 3) Share (URL + QR + codes)

After you click **Create Secure Link**, you get:

- **Copyable link**
- **QR code**
- **Read code** (open)
- **Modify code** (manage link later)
- Actions like **Change file** and **Access records** (product dependent)

![Result screen: link, QR, read/modify codes](/maipdf2026/Result_of_qr_link.png)

## Quick defaults that keep sharing professional

- Use a real **expiration** for external audiences.
- Set an **access limit** that matches the intended review round.
- Enable **email verification** when you need verified recipients.
- Use Telegram alerts only when your setup allows it.
