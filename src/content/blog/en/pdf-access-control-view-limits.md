# PDF Access Control: Set View Limits to Protect Your Documents

A view limit is the simplest and most effective access control you can apply to a shared PDF. It answers one question: how many times should this document be accessible?

![Settings with View Limits](/maipdf2026/MaiPDF_settings_expiration_telegram.png)

## Why View Limits Matter

Without a view limit, a shared link is essentially permanent. Anyone who discovers the URL — whether through forwarding, browser history, or a data breach — can access the document indefinitely. A view limit puts a ceiling on exposure.

**After the limit is reached**, the link returns an expiration notice. The document is no longer accessible to anyone, regardless of who has the URL.

## How to Set View Limits

On [MaiPDF](https://www.maipdf.com):

1. Upload your PDF
2. Open the settings panel
3. Set "Open Limit" to your desired number
4. Save and share the link

![Control Panel](/maipdf2026/how_to_control_panel.png)

## Choosing the Right Limit

The right number depends on your audience size and how they'll access the document.

| Scenario | Audience | Recommended Limit |
|----------|----------|-------------------|
| One-on-one proposal | 1 person | 10–15 views |
| Team review | 5–10 people | 50–100 views |
| Client presentation | 20–50 attendees | 200–500 views |
| Public resource | Unlimited | No limit |

**Why give more views than audience size?** Page refreshes, device switches, and re-reads all count as separate views. A single person reading a 3-page document might generate 3–5 view counts in one session.

![Open Limit Control](/maipdf2026/show_off/openlimit.png)

## View Limits + Other Controls

View limits work best when combined with other controls:

| Combination | Effect |
|------------|--------|
| View limit + expiration | Double protection — either trigger disables access |
| View limit + watermark | Limited access + identity tracking |
| View limit + download off | Readers can only view, and only a limited number of times |
| View limit + email verification | Each verified viewer consumes views from the shared pool |

## Monitoring View Count Usage

After sharing, check the access records to see how the view count is being consumed:

- Is it climbing faster than expected? Someone may be sharing the link widely.
- Is it barely used? Your audience may not have received the link.
- Is it exhausted too quickly? You may need to create a new link with a higher limit.

![Access Records](/maipdf2026/page_redirect_to_accessRecord.png)

## Common Pitfalls

**Setting limits too tight** — if you share with 10 people and set a limit of 10, the first person who refreshes the page locks out the last person. Always provide margin.

**Forgetting mobile behavior** — mobile browsers sometimes reload pages when switching apps. This consumes views even if the reader hasn't finished.

**Not communicating the limit** — if your audience needs to access the document multiple times (reference material, study guides), let them know the link has limited availability so they save important notes.

## When NOT to Use View Limits

- **Public marketing materials** — you want maximum reach, not restricted access
- **Documentation and guides** — reference materials should be always-available
- **Archived content** — if it's meant to be permanent, don't limit it
