# Document Distribution Compliance: A Checklist You Can Actually Use

<div class="intro-panel">
  <p>Compliance becomes easy when it’s broken into capabilities you can verify: who accessed, under what rules, and whether you can prove it later.</p>
</div>

![Compliance framework](/diagram/en/compliance-framework.svg)

## The 4 compliance capabilities

<div class="feature-section">
  <div class="feature-content">
    <h3>Identity & authorization</h3>
    <ul>
      <li>Use verification for sensitive documents</li>
      <li>Separate internal roles vs external recipients</li>
      <li>Apply “least privilege” by default</li>
    </ul>
  </div>
  <div class="feature-content">
    <h3>Limits & boundaries</h3>
    <ul>
      <li>Expiration dates (avoid permanent access)</li>
      <li>View limits (reduce leak window)</li>
      <li>Download/print restrictions when necessary</li>
    </ul>
  </div>
</div>

<div class="feature-section">
  <div class="feature-content">
    <h3>Audit logs</h3>
    <ul>
      <li>Access events (time, count, basic environment signals)</li>
      <li>Exports for reviews and investigations</li>
      <li>Alerting for unusual patterns</li>
    </ul>
  </div>
  <div class="feature-content">
    <h3>Governance</h3>
    <ul>
      <li>Policy templates by document type</li>
      <li>Change control for high-risk materials</li>
      <li>Retention rules for logs and documents</li>
    </ul>
  </div>
</div>

## A quick start checklist

- [ ] Verification enabled for confidential docs
- [ ] Expiration date set
- [ ] View limit set (if appropriate)
- [ ] Access logs enabled + exportable
- [ ] Owner and policy template recorded
