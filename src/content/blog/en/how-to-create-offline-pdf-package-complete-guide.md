## What is an Offline PDF Package?

**Definition:**
An offline PDF package converts your PDF into a standalone HTML file that:
- Can be opened in any modern browser
- Works without internet connection
- Maintains access controls (view limits, expiration)
- Prevents download, printing, and copying
- Uses local storage for permission checks

**Key Features:**
- ✅ Offline viewing capability
- ✅ View count limits
- ✅ Expiration dates
- ✅ Download prevention
- ✅ Print prevention
- ✅ Copy prevention
- ❌ No access tracking (online feature only)
- ❌ No dynamic watermarks (online feature only)

## Step-by-Step: Creating Offline PDF Package

### Step 1: Access Offline Mode

![Offline MaiPDF Home Page](/offlinepages/offline-MaiPDF-Home-Page.png)

**Getting Started:**
1. Navigate to MaiPDF platform
2. Access "Manage DRM → Offline" section
3. Enter offline mode interface
4. Ready to upload PDF

**Platform Access:**
- Free for personal use
- No registration required
- Instant access
- Simple interface

### Step 2: Upload Your PDF

![Upload Section Offline MaiPDF](/offlinepages/upload_section_offline_maipdf.png)

**Upload Process:**

1. **Select PDF File**
   - Click "Upload PDF" button
   - Browse your computer
   - Select PDF file to convert
   - File size limit: ≤ 100 MB (free tier)

2. **Upload File**
   - File is chunked and uploaded
   - Encrypted during upload
   - Processed on server
   - Wait for upload completion

**Upload Tips:**
- ✅ Ensure file is valid PDF
- ✅ Check file size (≤ 100 MB)
- ✅ Stable connection recommended
- ✅ Wait for upload confirmation

**What Happens:**
- File converted to HTML format
- Converted to HTML format
- Prepared for package generation

### Step 3: Configure Security Settings

![Security Setting](/offlinepages/security_setting.png)

**Configure Permissions:**

**View Limits:**
1. Set total view count
   - Maximum number of times file can be opened
   - Example: 5, 10, 20 views
   - Auto-locks when limit reached

2. Set session duration (optional)
   - Maximum viewing time per session
   - Time-based restrictions
   - Session timeout

**Expiration Date:**
1. Enable expiration date
2. Choose date and time
3. File becomes inaccessible after expiration
4. Automatic lock

**Action Restrictions:**
1. **Download Prevention**
   - Toggle "Prevent downloads" ON
   - Blocks file saving
   - Prevents extraction

2. **Print Prevention**
   - Enable "Prevent printing"
   - Blocks print functionality
   - No physical copies

3. **Copy Prevention**
   - Enable "Prevent copying"
   - Blocks text selection
   - Prevents content extraction

**Important Notes:**
- ❌ No access tracking available (online feature only)
- ❌ No dynamic watermarks available (online feature only)
- ✅ View limits work offline
- ✅ Expiration dates work offline
- ✅ Restrictions work offline

### Step 4: Generate Offline Package

![Result Download Zip File](/offlinepages/result_download_zip_file.png)

**Package Generation:**

1. **Review Settings**
   - Check all configurations
   - Verify security settings
   - Confirm view limits
   - Review expiration dates

2. **Generate Package**
   - Click "Generate Package"
   - Wait for processing
   - Package created on server
   - Ready for download

3. **Download Package**
   - Download ZIP file
   - Contains index.html + assets folder
   - Complete offline package
   - Ready for distribution

**Package Contents:**
- `index.html` - Main viewing file
- `assets/` - Resources and styles
- Permission scripts
- Local storage validation
- Network verification scripts

### Step 5: Distribute Package

**Distribution Methods:**

**Direct Distribution:**
- Send ZIP file via email
- Share via cloud storage
- Transfer via USB drive
- Include in product packaging

**Online Distribution:**
- Upload to your server
- Share download link
- Host on website
- Cloud storage sharing

**Best Practices:**
- ✅ Compress before emailing
- ✅ Use secure transfer methods
- ✅ Include usage instructions
- ✅ Test package before distribution

### Step 6: Recipients Open Package

![Click HTML Inside Zip to View](/offlinepages/click_html_inside_zip_to_view.png)

**Opening Process:**

1. **Extract ZIP File**
   - Recipients extract ZIP
   - Get index.html + assets folder
   - No installation needed

2. **Open HTML File**
   - Double-click index.html
   - Opens in default browser
   - Network verification required (first time)

3. **Network Verification**
   - Initial network connection required
   - Verification process completes
   - After verification, can view offline
   - Logs sync when device is online

4. **View PDF**
   - PDF rendered in browser
   - All restrictions active
   - View count tracked locally
   - Expiration checked locally

**Requirements:**
- Modern browser (Chrome, Edge, Safari, Firefox)
- JavaScript enabled
- Local storage enabled
- Network connection for initial verification
