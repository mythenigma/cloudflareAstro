# 用链接分享设计作品集：比发附件更专业，也更好控制

设计师发作品集，最怕的往往不是发不出去，而是发出去之后体验很差：打开慢、手机里难看、版本一多就乱、作品还没谈合作就先被完整转走。

![作品集分享应该先解决体验，再谈控制](/maipdf2026/show_off/goodlookingshow.png)

## 一眼看懂

| 如果你想要…… | 用链接更合适，因为…… |
|------|------|
| 第一眼更专业 | 打开像正式展示，不像在收附件 |
| 手机里也能顺着看 | 对方不用先下载再找软件 |
| 一个稳定入口 | 更新作品时不容易版本分叉 |
| 按场景控制开放程度 | 求职、提案、竞标可以分开处理 |

## 最常见的 4 个场景

| 场景 | 最在意什么 |
|------|------|
| 求职投递 | 打开快、展示完整、版本清楚 |
| 客户提案 | 既好看，又别太容易被直接转走 |
| 竞标 / 高敏感展示 | 可看，但要控制传播 |
| 名片 / 活动现场 | 二维码一扫就能进入作品集 |

## 常见起步配置

| 场景 | 打开次数 | 有效期 | 下载 | 备注 |
|------|------|------|------|------|
| 求职投递 | 20-50 | 14-30 天 | 可选 | 重点是低摩擦 |
| 客户提案 | 5-20 | 7-14 天 | 建议关 | 适合保护方案和案例 |
| 高敏感展示 | 低 | 短 | 关 | 水印通常值得开 |

![同一个作品集，也可以按不同场景选择不同分享策略](/maipdf2026/show_off/shareandsend.png)

## 更专业的分享流程

| 步骤 | 更好的做法 |
|------|------|
| 导出干净 PDF | 先把封面、顺序、节奏理顺 |
| 上传生成入口 | 把作品集变成正式阅读链接 |
| 按场景补最少设置 | 不要一上来全锁死 |
| 发一个主链接或二维码 | 让对方直接进入展示 |
| 更新时优先保留原入口 | 不要每次都发出新版本 |

![上传和设置是作品集链接分享的标准入口](/maipdf2026/Maipdf_LANDING_PAGE.png)

## 最值得用的控制

![对高风险展示，阅读控制和展示体验要一起考虑](/maipdf2026/show_off/viewercontainer_noprint_nodownlaod.png)

| 控制 | 什么时候用 | 为什么 |
|------|------|------|
| 下载控制 | 发给陌生客户、竞标对象、敏感展示 | 阅读不等于让文件立刻落地 |
| 打开次数 | 小范围、阶段性展示 | 不让入口长期漂在外面 |
| 水印 | 担心截图传播或未授权转发 | 比空说“禁止外传”更实际 |

## 常见错误

| 错误 | 更好的做法 |
|------|------|
| 先发一个大附件 | 先给一个正式展示入口 |
| 每改一次就发新链接 | 作品集更新时尽量保留主入口 |
| 求职场景也限制太重 | 先保证顺畅，再补轻控制 |
| 完全不考虑手机端 | 默认很多第一次打开都发生在手机上 |
