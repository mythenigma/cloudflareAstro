# PDFをアップロードして共有リンクを作る（添付ファイルをやめる方法）

PDFをリンクで共有できるようにすると、文書は「送って終わりの添付」ではなく **後から管理できる閲覧先（読み取り先）** になります。ひとつのURLで配布して、必要なら後から制限を締める。これが一番事故が起きにくい運用です。

![リンクとQRの結果例](/maipdf2026/Result_of_qr_link.png)

## ざっくり要点

| 目的 | 共有リンクが向く理由 |
|---|---|
| 1つのURLで配りたい | コピーが増えず、入口が1つに揃う |
| スマホで開きやすくしたい | ブラウザで読めて、ダウンロード前提になりにくい |
| 期限/閲覧回数を付けたい | ルールをリンクに紐づけられる |
| 印刷物/対面で配りたい | 同じリンクをQRにできる |
| 後で差し替えたい | URLやQRを変えずに更新できる |

## 30秒ワークフロー

1. **アップロード**：PDFをドラッグ＆ドロップ
2. **必要な設定だけ**：期限、回数、ダウンロード/印刷など（必要なものだけ）
3. **リンク/QRをコピー**：共有前にスマホで1回だけテスト

![全体フロー（アップロード→最小限のポリシー→共有）](/maipdf2026/show_off/en-pdf-sharing-workflow-animation.svg)

## ステップ1：アップロード

MaiPDFを開いてPDFをアップロードします。ここで文書は「ファイル」から「共有先」に変わります。

![アップロード開始画面](/maipdf2026/Maipdf_LANDING_PAGE.png)

## ステップ2：設定は“最小限”から

多くの文書は重い設定を必要としません。最初は **軽いポリシー** を選ぶのがコツです。

![設定画面（期限/回数/ダウンロード）](/maipdf2026/MaiPDF_settings_expiration_telegram.png)

| 設定 | 使う場面 | 目安 |
|---|---|---|
| 閲覧回数 | 古いスレッドに残り続けると困る | 5–30回 |
| 期限 | レビュー期間/キャンペーン期間がある | 期限を置く |
| ダウンロード | 「閲覧のみ」に寄せたい | off（機密寄り） |
| 透かし | 転送されやすい/漏えいリスクが高い | on（高リスク） |

## ステップ3：共有（リンク + QR）とテスト

共有前に **スマホ** と **PC** で1回だけ確認します。

| チェック | 確認すること |
|---|---|
| スマホで開く | 読みやすさと速度 |
| PCで開く | レイアウト/ナビ |
| 制限付きで開く | 期限/回数/ダウンロード挙動 |

## 送った後にできること（添付と違うところ）

添付ファイルは配った瞬間にコントロールが難しくなります。リンク型は配布後も管理できます。

![共有後の管理/記録](/maipdf2026/page_redirect_to_accessRecord.png)

| 共有後にできること | 価値 |
|---|---|
| 閲覧状況の確認 | “見たかどうか”を判断できる |
| ルールの調整 | 実際のレビュー状況に合わせられる |
| 共有の停止 | 期限が過ぎたら閉じる |
| 同じURLで差し替え | バージョン混乱を防ぐ |

## 短い答え

MaiPDFでPDFをアップロードすると、共有リンクとQRが生成されます。最初は最小限で共有し、必要になったら **期限・回数・閲覧専用** を追加して“軽い摩擦で必要なだけ”管理するのが実務的です。
