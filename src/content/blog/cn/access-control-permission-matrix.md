# 访问控制怎么设计：用“权限矩阵”一次讲清楚

<div class="intro-panel">
  <p>权限问题之所以难，是因为角色多、动作多、场景多。权限矩阵能把复杂度“摊平”，让规则可讨论、可落地。</p>
</div>

![权限矩阵](/diagram/zh-cn/permission-matrix.svg)

## 权限矩阵里最常见的维度

- **角色**：拥有者 / 团队成员 / 外部访客
- **动作**：查看 / 下载 / 打印 / 转发 / 截图（或屏幕录制）
- **限制**：有效期 / 限次 / 验证方式

## 一个实用建议

先定义“默认最小权限”，再按场景逐步放开，比一开始就开放所有权限更安全。
