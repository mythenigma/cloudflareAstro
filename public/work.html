⚠️ 核心问题诊断

问题 1：Stub / 空壳文章 — 严重影响 SEO
101 篇博文只有标题 + 一句话描述，没有实质内容。搜索引擎判定为薄内容 (thin content)，拖累整站质量分数。
问题 2：重复 CTA 块
布局 (BlogPost.astro) 自动在文章底部插入 "Try MaiPDF Now" CTA，但多篇文章内嵌了自己的 conclusion-box / cta-section，导致两个蓝色大块叠在一起。
问题 3：文章内嵌 <style> 冲突
多篇文章有独立的 <style> 块（蓝色主题 vs 紫色全局），样式互相覆盖，视觉不统一。
问题 4：缺少 ToC / 阅读时间
长文（2000+ 字）没有目录和阅读时间估算，不符合专业博客标准。

🧠 认知 & 注意事项 (Lessons Learned)

1. 重复 CTA 机制
BlogPost.astro 第 68-73 行通过 showDefaultCta（默认 true）自动在文章底部插入 CTA。写新文章时 不要 再手动加 conclusion-box / cta-section，否则又会出现双 CTA。如果某篇文章需要自定义 CTA，在 frontmatter 中设 showDefaultCta: false。
2. 文章内 <style> 块冲突
多篇老文章有独立 <style> 块（蓝色主题），与全局 blog-post.css 的紫色主题冲突。新文章应 只使用 已有 CSS class（.intro-panel, .feature-card, .steps-container, .comparison-table 等），不要写内联 <style>。
3. Show-off 图片仅适用于 MaiPDF 文章
/maipdf2026/show_off/ 目录的 24 张营销图都是 MaiPDF 品牌的。MaiImg 文章应使用 /maiimg/ 目录的图片（maiimg-head.png, image_result.png 等），不要混用。
4. 图片路径规范
✅ /maipdf2026/ — 当前统一路径
❌ /maipdf-images/ — 旧路径，已全部迁移，不要使用
✅ /maiimg/ — MaiImg 产品图
✅ /diagram/{lang}/ — SVG 流程图（按语言分目录）
5. Git 基线
原始无损文章基线：commit 89c7db2。如果以后需要恢复文章，从此 commit 检出。
