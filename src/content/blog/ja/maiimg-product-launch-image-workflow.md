# Maiimgで実現するプロダクトローンチ画像ワークフロー

<div class="intro-panel">
  <p>新商品の発表日はマーケ・営業・PR・代理店が同じビジュアルを同時に使う必要があります。<strong>Maiimg</strong>なら25枚までの画像を一括アップロードし、閲覧回数や有効期限を制御しながら、各チームにリンク/QRコードで配布できます。</p>
</div>

![Maiimg Launch Gallery](/maiimg/maiimg-head.png)

## 典型的な課題

- Slackやメールで複数バージョンが飛び交う
- 代理店が古いKVを使い続ける
- ダウンロード履歴が追えずコンプライアンスが曖昧
- 急な差し替えで全リンクを再送する手間

## ロール別ギャラリー設計

| 対象 | 内容 | 推奨設定 |
|------|------|----------|
| ソーシャルチーム | 1:1 / 9:16 KV、ストーリー用 | 閲覧150回、14日、有効・DL OFF |
| 営業/セールス | 提案資料、ライフスタイル写真 | 80回、30日、DL ON |
| 小売・流通 | パックショット、棚割り画像 | 200回、45日、DL ON + QR |
| PR/メディア | ロゴ、製品切り抜き | 40回、7日、パスワード + DL ON |

## セットアップ手順

1. **アップロード**：ギャラリーごとに最大25枚。用途別に複数ギャラリーを作成
2. **権限設定**：閲覧回数・有効期限・ダウンロード可否・セッション時間（必要に応じて）
3. **リンク/QR発行**：各ギャラリーで共通URLとQRを自動生成
4. **配布**：Notion、Slack、メールテンプレ、印刷物に組み込み
5. **モニタリング**：ビュー数、人気画像、地域データを確認し、利用状況を可視化

## 例：配信テンプレ

```
件名：Launch X 公式アセット（社内限定）
Social: https://maiimg.com/...（12/15まで）
Sales: https://maiimg.com/...（DL可）
Retail用QR: 添付PDFを印刷して店頭で掲示してください。
```

## ベストプラクティス

- ギャラリー名に日付を入れて差分管理
- Embargo前は閲覧回数10・有効期限3日で配布し、ローンチ時に本番ギャラリーへ切替
- 店舗/代理店向けにQRコード入りカードを同梱
- 終了後はギャラリーを削除して旧素材を排除

## 解析で得られる示唆

- どのチームが未アクセスか → リマインド
- 最も閲覧された画像 → 広告クリエイティブの優先順位化
- ピーク時間帯 → リリースや投稿タイミングの最適化

## まとめ

Maiimgを使えば、新商品ローンチの画像配布を「一元管理しつつ柔軟に分配」できます。アクセス制御とQR配布を組み合わせ、スピードとブランド統制を同時に実現しましょう。
