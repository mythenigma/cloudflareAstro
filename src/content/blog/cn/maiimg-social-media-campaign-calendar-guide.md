# Maiimg社交媒体活动图片内容日历指南

<div class="intro-panel">
  <p>一个活动同时在抖音、小红书、微博、Instagram上线时，图片版本最容易混乱。利用<strong>Maiimg + 内容日历</strong>的组合，可以把每波素材提前打包，分平台设置访问权限，并用链接/二维码同步给所有内容产出人。</p>
</div>

![Maiimg Social Calendar](/maiimg/maiimg-head.png)

## 步骤总览

1. 在内容日历中拆分波次（预热、正式、复盘）
2. 每个波次创建一个Maiimg图库，最多25张
3. 在图库说明中写明发布日期、话题、话术
4. 设置查看次数/到期日，防止错误发布
5. 把链接和二维码嵌入到日历、飞书文档或Notion

## 平台拆分建议

| 平台 | 素材类型 | 权限策略 |
|------|----------|----------|
| 抖音/视频号 | 9:16 封面、竖图 | 查看150次，14天，下载关闭 |
| 小红书 | 4:5 图文、长图模板 | 查看120次，21天，下载开启 |
| 微博/Instagram | 1:1 单图、多图排版 | 查看200次，30天，下载开启 |
| KOL/合作方 | 品牌KV、Logo、注意事项 | 查看80次，7天，密码保护 |

## 内容日历嵌入示例

```
周一：预热短视频
- 素材图库：https://maiimg.com/...
- 文案：见飞书《预热文案 V2》
- Tag：#品牌挑战 #活动话题
```

## 二维码应用

- 给外包团队或线下门店张贴“扫码获取本周素材”
- 直播间主持人手边放置打印二维码，随时更新封面
- 线下活动签到台提供“社媒素材包”卡片

## 数据追踪

- 每个图库的查看次数 = 对应波次的执行度
- 未打开的团队 = 需要提醒或补训练
- 最受欢迎的图片 = 下次活动的优先创意
- 地域/设备数据 = 判断是否有海外团队访问

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 素材版本不一致 | 在图库中注明“当前版本号”，旧图立即删除 |
| 超过25张 | 分波次或分平台建多个图库 |
| 链接过期忘记 | 提前2天设置提醒，自动延长 |
| 外部合作方乱发 | 限制查看次数并开启密码，必要时随时删除图库 |

## 结论

Maiimg让社交媒体内容日历真正“落地”：

- 每波素材独立控制，避免误发
- 链接+二维码覆盖线上线下
- 通过数据知道团队是否按计划执行

把Maiimg当作内容日历的“图片交付模块”，社交媒体协作效率会显著提高。
