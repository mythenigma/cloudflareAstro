# PDF 공유 링크에서 다운로드를 비활성화하기(보기 전용)

첨부파일은 상대가 열자마자 로컬 복사본이 생겨 이후 통제가 어렵습니다. **보기 전용 링크**는 브라우저에서 페이지를 모두 읽을 수 있지만, **다운로드 버튼이 제거**(필요하면 인쇄도 차단)되어 파일이 남기 쉬운 경로를 줄입니다.

![보기 전용 뷰어(다운로드 없음)](/maipdf2026/show_off/viewercontainer_noprint_nodownlaod.png)

## MaiPDF에서 보기 전용 링크 만드는 방법

1. [maipdf.com](https://maipdf.com) 에서 PDF 업로드
2. 설정에서 **다운로드 비활성화**(필요 시 인쇄도 비활성화)
3. **열람 횟수 제한**과 **만료(유효기간)** 설정
4. 링크 생성 후 공유

![설정 패널(다운로드/만료/횟수)](/maipdf2026/MaiPDF_settings_expiration_telegram.png)

## 수신자 경험은 어떻게 보이나?

- **페이지는 정상 열람**: 스크롤/줌/페이지 이동
- **다운로드 버튼 없음**: 저장/다운로드 아이콘 제거
- **인쇄 차단 가능**: 필요할 때만
- **원본 파일을 “받는” 흐름 감소**: 로컬 보관을 전제로 하지 않음

대부분의 수신자는 다운로드가 없다는 걸 의식하지 않고 그냥 문서를 읽습니다.

## 다운로드를 꺼두면 좋은 상황

| 상황 | 다운로드 off | 이유 |
|---|---:|---|
| 제안서(협상 중) | 예 | 무단 복사/재배포 감소 |
| 계약서 검토 | 예 | 버전 관리(최신본만 노출) |
| 유료 리포트/프리미엄 콘텐츠 | 예 | “파일”이 아니라 “접근” 제공 |
| 디자인 초안 | 예 | 작업 중 산출물 보호 |
| 홍보 브로슈어 | 아니오 | 널리 배포가 목적 |
| 서명용 문서 | 아니오 | 인쇄가 필요할 수 있음 |

## 다른 제어와 함께 쓰면 더 강해진다

다운로드 off는 “가벼운 재배포”를 줄입니다. 더 강하게 하려면:

- **열람 횟수 제한**: N회 이후 링크 종료
- **만료**: 지정 날짜에 자동 종료
- **워터마크**: 유출 억제(식별 정보 표시)
- **이메일 인증**: 열기 전에 본인 확인

## 공유 후에도 언제든 변경 가능

이미 링크를 보냈더라도 [Control Center](https://www.maipdf.com/6/control-center.html) 에서 다운로드 ON/OFF를 바꿀 수 있습니다. 다음 열람부터 즉시 적용되며, 링크를 다시 보낼 필요가 없습니다.
