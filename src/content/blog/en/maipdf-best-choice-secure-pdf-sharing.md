# MaiPDF: A Practical Tool for Controlled PDF Sharing

<div class="intro-panel">
  <p>MaiPDF is built for one thing: sharing PDFs with control over who can open them, how many times, and for how long. No app installs, no signup required for the reader.</p>
</div>

![MaiPDF landing page — upload and share immediately](/maipdf2026/Maipdf_LANDING_PAGE.png)

## What you get in 60 seconds

Upload a PDF → set your rules → get a link. That's the whole workflow.

The reader clicks the link and sees the PDF directly in a browser viewer. No download prompt, no login screen, no "please install our app" pop-up.

![Clean viewer — the reader sees the PDF, nothing else](/maipdf2026/show_off/showpdf.png)

## Controls you can set on each link

| Control | What it does | When to use it |
|---------|-------------|---------------|
| **Open limit** | Link stops working after N opens | Proposals, quotes — limit circulation |
| **Expiry date** | Link dies after a set date | Campaign materials, time-sensitive docs |
| **Download off** | No save/download button in viewer | Protecting IP, draft reviews |
| **Print off** | No print option | Confidential internal docs |
| **Email verification** | Reader must enter email + code | Track exactly who opened |
| **Watermark** | Your text burned into every page | Discourage screenshots |
| **Telegram alerts** | Push notification on every open | Know instantly when someone reads it |

All of these are set in one panel before you create the link. No extra steps, no separate settings page.

![Settings panel — open limit, expiry, email verification, Telegram alerts](/maipdf2026/MaiPDF_settings_expiration_telegram.png)

## What happens after you share

Every open is logged. You get:

- **Who opened** (if email verification is on) or how many opens (if not)
- **When** — timestamp for each view
- **How many times** — total opens per link

Check everything in [Control Center](https://www.maipdf.com/6/control-center.html) anytime.

![Access records — who, when, how many times](/maipdf2026/page_redirect_to_accessRecord.png)

### Change settings after sharing

Already sent the link? You can still:
- Lower the open limit
- Change the expiry date
- Turn off access entirely (revoke)
- Swap the PDF file behind the same link (the URL doesn't change)

![Update settings on an existing link](/maipdf2026/swap_file_easy_in_user_control_panel.png)

## Where MaiPDF fits best

**Sales & proposals** — share a quote with open limit, get a Telegram alert when the client reads it, follow up at the right moment.

**Client reviews** — send a design draft with download disabled and watermark on. The client can view and give feedback; they can't save and forward the raw file.

**Internal distribution** — HR docs, policy updates, training materials. Set email verification + company domain allowlist so only team members can open.

**Education** — course materials with expiry dates so last semester's students can't access this term's content.

## Where it's not the right tool

- **Public downloads** — if you want anyone to download freely, just host the PDF on your server or use a CDN.
- **Permanent archives** — MaiPDF is designed for controlled, time-scoped sharing. For long-term public storage, a static host is simpler.
- **Heavy enterprise compliance** — if you need SOC 2 audit trails or custom SSO integration, you'll need a full DMS (document management system).
