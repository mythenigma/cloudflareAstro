# How to Update or Modify a Shared PDF Link

<div class="intro-panel">
  <p>You've shared a PDF link and now you need to make a change — the file was updated, the expiry needs extending, or you want to revoke access entirely. You don't need to send a new link. MaiPDF lets you make all of these changes from Control Center while the same URL stays valid.</p>
</div>

## What you can change after sharing

| Action | What changes | Link URL changes? |
|--------|-------------|------------------|
| **Replace file** | PDF content the reader sees | No — same URL |
| **Change expiry** | When the link stops working | No |
| **Adjust view limit** | How many more opens are allowed | No |
| **Toggle download** | Whether reader can download | No |
| **Enable watermark** | Shows reader's ID on pages | No |
| **Revoke link** | Link immediately goes inactive | No |

None of these require resending. Everyone who has the original URL automatically gets the updated behavior next time they open it.

## How to replace the file behind an existing link

This is the most common need: you updated the document and want everyone to see the new version.

1. Go to [Control Center](https://www.maipdf.com/6/control-center.html)
2. Find the link in your dashboard
3. Click **Replace File**
4. Upload the new PDF

Done. The link URL stays identical. Anyone who clicks it from now on gets the updated version.

![Control Center — all management options in one place](/maipdf2026/user_control_panel_alotof_functions.png)

## How to change access settings

From the same Control Center dashboard:

- **Extend expiry** — click the expiry date and pick a new one
- **Raise/lower view limit** — useful if more people need access than you planned
- **Toggle download** — switch from view-only to downloadable (or back) at any time
- **Add watermark** — enable it retroactively if accountability becomes important

![Settings panel reflected in Control Center for active links](/maipdf2026/MaiPDF_settings_expiration_telegram.png)

## How to revoke a link immediately

If you need to stop access for any reason — the document was shared by mistake, the deal fell through, the recipient shouldn't have access anymore — use **Revoke** in Control Center. The link becomes inactive immediately. Anyone who tries to open it sees an "access revoked" message.

## The recommended workflow

For document series or ongoing materials:

1. **Upload and create your link once**
2. **Share that link** — in email, in a portal, or in a QR code
3. **Update the file as needed** using Replace File (link never changes)
4. **Adjust access rules** over time as the document's sensitivity changes
5. **Revoke** when the document's lifecycle ends

This pattern is cleaner than creating a new link every revision cycle. It avoids the "which version did they see?" version confusion problem.
