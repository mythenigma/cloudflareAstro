# 网络验证协议概览

<div class="intro-panel">
  <p>当你分享一个受保护的 PDF 链接，接收者打开时会经历哪些验证？这些验证如何保护你的文件？</p>
</div>

## 验证层级

MaiPDF 支持多层验证，可以根据安全需求组合使用：

| 验证方式 | 作用 | 适用场景 |
|---------|------|---------|
| 密码验证 | 知道密码才能打开 | 基础保护 |
| 邮箱验证 | 输入邮箱后发验证码 | 需要追踪身份 |
| 打开次数 | 超过限制自动锁定 | 限量分发 |
| 有效期 | 过期后链接失效 | 时效性文件 |
| SecureView | 防下载/打印/截屏 | 机密内容 |

## 验证流程

接收者打开链接后的典型流程：

1. **链接有效性检查** → 是否过期、是否超限
2. **密码验证**（如开启）→ 输入密码
3. **邮箱验证**（如开启）→ 输入邮箱 → 收验证码 → 输入验证码
4. **渲染文档** → 应用 SecureView/FenceView/水印等保护
5. **记录访问日志** → 时间、IP、设备

## 邮箱验证的工作原理

邮箱验证是目前最推荐的身份验证方式：

- 查看者输入邮箱地址
- 系统发送一次性验证码
- 输入正确验证码后才能查看
- 每次验证都会记录

优势：不需要接收者注册账号，又能准确识别身份。

## 安全日志

每次访问都会产生日志记录：

| 记录项 | 说明 |
|--------|------|
| 访问时间 | 精确到秒 |
| IP 地址 | 查看者的网络地址 |
| 邮箱 | 如果开启了邮箱验证 |
| 设备 | 浏览器和操作系统 |
| 查看时长 | 停留了多久 |

## 常见搭配方案

| 安全级别 | 推荐配置 |
|---------|---------|
| 基础 | 密码 |
| 标准 | 邮箱验证 + 打开次数限制 |
| 高级 | 邮箱验证 + SecureView + 动态水印 |
| 最高 | 邮箱验证 + FenceView + 动态水印 + 有效期 |

## 总结

MaiPDF 的验证体系从链接层到内容层层层保护。根据文件敏感程度选择合适的验证组合，安全不用复杂化。
