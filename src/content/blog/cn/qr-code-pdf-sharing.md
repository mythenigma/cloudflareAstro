# 二维码分享 PDF：从扫码即看到线下分发，怎么做更顺

二维码分享 PDF 的价值，不是多了一张码图，而是把“复制链接、输入网址、找附件”这些动作，压缩成了一次扫码。

![二维码最适合把线下接触点延伸成完整阅读入口](/maipdf2026/show_off/share%20pdf%20wordwide.png)

## 一眼看懂

| 如果场景是…… | 二维码更合适，因为…… |
|------|------|
| 海报、宣传册、桌牌 | 不需要输入长链接 |
| 名片、简历卡、作品集卡片 | 一扫就能看完整资料 |
| 包装、说明页、标签 | 小面积印刷也能承接长文档 |
| 展会、课堂、活动现场 | 很多人能同时快速进入同一份 PDF |

## 最适合放二维码的地方

| 放置位置 | 扫码后最适合看到什么 |
|------|------|
| 海报 / 宣传册 | 放不下的完整资料 PDF |
| 名片 / 简历卡 | 作品集、简历、公司介绍 |
| 包装 / 说明页 | 手册、说明文档、合规资料 |
| 展位 / 课堂物料 | 产品资料、讲义、活动包 |

## 实际流程很简单

| 步骤 | 你在做什么 |
|------|------|
| 上传 PDF | 先建立可控阅读链接 |
| 生成二维码 | 二维码只是同一链接的另一种入口 |
| 放到合适场景 | 在线下或半线下触点承接阅读 |
| 上线前自测 | 确认真实手机扫码和阅读体验 |

![二维码分发更像是把已有阅读入口延伸到线下](/diagram/zh-cn/qrcode-share-3steps.svg)

## 上线前重点测什么

| 测试项 | 为什么要测 |
|------|------|
| iPhone 和 Android 都能扫开 | 真实扫码表现会有差异 |
| 手机里读起来顺不顺 | 扫得快不代表看得舒服 |
| 限次 / 到期 / 验证是否正常 | 二维码同样继承这些规则 |
| 真实距离能不能扫到 | 工位上能扫开，不等于现场也行 |

## 哪些控制仍然有效

| 控制项 | 二维码访问时是否仍然有效 |
|------|------|
| 打开次数 | 有效 |
| 有效期 | 有效 |
| 下载控制 | 有效 |
| 水印 | 有效 |
| 验证流程 | 有效 |

这点很关键，因为它意味着你可以把二维码用于线下分发，同时保留线上访问控制。

## 二维码排版时最该注意什么

| 规则 | 更好的做法 |
|------|------|
| 尺寸 | 宁可大一点，也不要为了排版压太小 |
| 对比度 | 黑白高对比最稳 |
| 留白 | 四周别挤满图形和文字 |
| 动作提示 | 写清楚“扫码查看完整资料”之类的说明 |

## 一个很值钱的优势：更新 PDF 不一定要重做二维码

| 这种情况很有用 | 为什么 |
|------|------|
| 菜单、手册、目录更新 | 不必把旧二维码全部推倒重来 |
| 作品集刷新内容 | 以后扫到的还是当前版本 |
| 活动物料印出去后又改资料 | 一个码还能继续承接新文件 |

## 直接回答这个页面

想用二维码分享 PDF，正确做法不是先做一张码，而是先在 MaiPDF 生成一个可控链接，再让二维码指向它。这样别人一扫就能看，你也还能保留限次、到期、下载控制，后面更新 PDF 时二维码也可以继续用。
