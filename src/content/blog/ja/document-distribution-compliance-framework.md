# 文書配布のコンプライアンス：本当に使えるチェックリスト

<div class="intro-panel">
  <p>コンプライアンスは「確認できる能力」に分解すると簡単です。誰が、どの条件でアクセスし、後から証明できるか——それだけです。</p>
</div>

![Compliance framework](/diagram/en/compliance-framework.svg)

## 4つのコンプライアンス能力

<div class="feature-section">
  <div class="feature-content">
    <h3>本人性 & 権限</h3>
    <ul>
      <li>機密文書は本人確認を必須にする</li>
      <li>社内ロールと外部受領者を分ける</li>
      <li>原則「最小権限」</li>
    </ul>
  </div>
  <div class="feature-content">
    <h3>制限 & 境界</h3>
    <ul>
      <li>有効期限（永久リンクを避ける）</li>
      <li>閲覧回数制限（流出ウィンドウを縮める）</li>
      <li>必要ならダウンロード／印刷制限</li>
    </ul>
  </div>
</div>

<div class="feature-section">
  <div class="feature-content">
    <h3>監査ログ</h3>
    <ul>
      <li>アクセスイベント（時刻、回数、基本的な環境情報）</li>
      <li>レビュー／調査用のエクスポート</li>
      <li>異常検知のアラート</li>
    </ul>
  </div>
  <div class="feature-content">
    <h3>ガバナンス</h3>
    <ul>
      <li>文書タイプ別のポリシーテンプレート</li>
      <li>高リスク文書の変更管理</li>
      <li>ログ／文書の保管ルール</li>
    </ul>
  </div>
</div>

## すぐ使えるチェックリスト

- [ ] 機密文書は本人確認を有効化
- [ ] 有効期限を設定
- [ ] 必要に応じて閲覧回数制限を設定
- [ ] アクセスログが有効でエクスポート可能
- [ ] 所有者とポリシーテンプレートを記録
