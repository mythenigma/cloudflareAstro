# PDF 访问控制的实操方案

<div class="intro-panel">
  <p>"访问控制"听起来很抽象，其实就是回答一个问题：<strong>谁能看、能看几次、能不能下载</strong>。</p>
</div>

## 四种常见场景的配置方案

### 场景一：给客户发报价单

**需求**：只想让这一个客户看，不想被转发。

| 配置项 | 设置 |
|--------|------|
| 打开次数 | 5 |
| 有效期 | 7 天 |
| 保护模式 | SecureView |
| 邮箱验证 | 关闭 |

### 场景二：团队内部分发会议纪要

**需求**：10 个人的团队都能看，但不要外泄。

| 配置项 | 设置 |
|--------|------|
| 打开次数 | 30 |
| 有效期 | 30 天 |
| 保护模式 | 不限制 |
| 邮箱验证 | 关闭 |

### 场景三：给投资人发财务报告

**需求**：绝对保密，看完即废，可追溯。

| 配置项 | 设置 |
|--------|------|
| 打开次数 | 3 |
| 有效期 | 3 天 |
| 保护模式 | FenceView |
| 邮箱验证 | 开启 |
| 动态水印 | 开启 |

### 场景四：公开分享产品手册

**需求**：越多人看越好，不限制。

| 配置项 | 设置 |
|--------|------|
| 打开次数 | 999 |
| 有效期 | 不设 |
| 保护模式 | 不限制 |
| 邮箱验证 | 关闭 |

- **查看统计**：已被打开几次
- **关闭链接**：紧急情况一键失效
- **替换文件**：更新内容，链接不变

## 总结

访问控制不需要研究复杂的权限系统。根据场景从上面四个方案里选一个，填好参数，点生成就行了。
