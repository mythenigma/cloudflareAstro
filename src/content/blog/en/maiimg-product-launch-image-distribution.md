## Step-by-Step Configuration

### 1. Bulk Upload Assets

- Drag-and-drop up to 25 images (PNG/JPG/WebP)
- Use naming conventions (e.g., `SKU_View_Angle`)
- Create separate galleries per audience if needed

### 2. Set Security Controls

- **View Limits**: 50â€“100 for marketing teams, 10â€“20 for agencies
- **Expiration**: Align with campaign phases (pre-launch, launch week, post-launch)
- **Download Rules**: Enable for retail partners, disable for embargoed assets
- **Time Controls**: 60-second viewing window for embargoed previews

### 3. Generate Link + QR Code

- Copy link for email distribution lists
- Download QR code for launch kits, press releases, showroom displays
- Embed QR code in PDF briefs or physical packets

### 4. Segment Distribution

| Audience | Delivery Method | Notes |
|----------|-----------------|-------|
| Marketing Team | Direct link + Slack announcement | High view limits, no password |
| Sales Team | Link embedded in launch brief | Enable downloads for pitch decks |
| PR Agency | Password-protected link | Add expiration after press day |
| Retail Partners | Printed QR + email link | Provide packshots + shelf visuals |
| Influencers | QR code on event invites | Track access to monitor engagement |

### 5. Monitor Access

- Watch total views to confirm engagement
- Identify which assets are most downloaded
- Detect early leaks via location/time patterns
- Disable access immediately if needed

## Best Practices

### Role-Based Galleries

- Create **Marketing**, **Sales**, **PR**, **Retail** galleries
- Tailor settings per role
- Update galleries independently during refreshes

### Embargo Management

- Use short expiration dates for pre-launch access
- Auto-expire galleries at launch to avoid outdated assets
- Replace with live galleries on launch day

### Localization Support

- Duplicate galleries by region
- Include localized packaging shots
- Provide region-specific QR codes
- Track access to confirm delivery

### Launch Retro Analysis

Use Maiimg analytics post-launch to measure effectiveness:

- Which teams accessed assets the most?
- Which images were most viewed/downloaded?
- Did view spikes align with campaign milestones?
- Were there unused assets to improve next time?
